# SESSION_STRATEGY.md — 세션 세분화 운영안

## 목표
- 메인 대화 반응속도 유지
- 긴 작업/무거운 작업의 분리 실행
- 80% 이전 선제 전환으로 지연 최소화

## 세션 분류

### 1) MAIN (대장님 대화 전용)
- 용도: 지시/결정/요약/승인
- 원칙: 짧고 빠르게, 실행 상세는 분리 세션으로 이관
- 전환: 80% 도달 시 즉시 /new

### 2) WEB-OPS (약품관리 페이지 유지보수)
- 용도: UI 수정, 모바일/PC 점검, 배포
- 산출물: 변경사항 요약 + 링크 + 전/후 차이

### 3) FILE-OPS (치부책/결혼식 파일 정리)
- 용도: 정리/인덱스/백업/검색 스크립트
- 산출물: 운영본 경로 + 백업 경로 + 검색 방법

### 4) CRON-OPS (자동화 운영)
- 용도: 크론 정리/장애 대응/주기 최적화
- 산출물: 유지/삭제/통합 제안서 + 적용 로그

## 실행 규칙
1. 대장님 요청 접수 후 MAIN에서 착수 보고
2. 3분 이상 걸리거나 파일/배포 작업이면 분리 세션 태그로 실행
3. 완료 시 MAIN으로 결과만 복귀 보고
4. 상태 보고 포맷: 착수 → 중간 → 완료

## 트리거 문구 (대장님용)
- "웹작업 모드" → WEB-OPS 우선
- "파일정리 모드" → FILE-OPS 우선
- "자동화 점검" → CRON-OPS 우선
- "긴작업 분리" → 해당 작업 즉시 분리 실행

## 쾌적성 가드레일
- MAIN 70%: 경고 없는 사전 안내
- MAIN 80%: 즉시 /new 권장 + 인계 파일 갱신
- MAIN 90%: 강제 전환 경고
